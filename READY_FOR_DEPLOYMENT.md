## ‚ö†Ô∏è IMPORTANT: Database Persistence

**WARNING**: Render's free tier does NOT persist files between deployments!

Your SQLite database will be **reset every time** you deploy. For production use:

### Recommended Solutions:

1. **Upgrade to Render Paid Plan** ($7/month)
   - Persistent disk storage
   - Always-on (no cold starts)
   - Better performance

2. **Use PostgreSQL** (Free tier available on Render)
   - Create PostgreSQL database
   - Update `DATABASE_URL` environment variable
   - Modify SQLAlchemy connection (minimal changes needed)

3. **Use External Database** (Recommended for production)
   - **Supabase**: Free PostgreSQL with 500MB storage
   - **PlanetScale**: Free MySQL with 5GB storage
   - **MongoDB Atlas**: Free MongoDB with 512MB storage

## üêõ Troubleshooting

### Common Issues

**Backend not starting**
- Check Render logs
- Verify Python version (3.11.0)
- Ensure all dependencies installed

**Frontend not connecting to backend**
- Verify `VITE_API_URL` is correct
- Check CORS settings
- View browser console for errors

**Database not found**
- Check build logs
- Verify `setup_and_seed.py` ran successfully
- Ensure `DATABASE_URL` is correct
